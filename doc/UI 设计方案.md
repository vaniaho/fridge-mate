# Smart Fridge Product-Grade GUI Design Scheme

根据需求，本次 GUI 设计方案将采用"桌面启动器 (Launcher) + 独立应用模块"的架构，确保极佳的扩展性和产品级的交互体验。

> **屏幕规格**：7 英寸 1024 × 600 横屏触摸显示屏。以下所有布局均基于横屏设计。

> \[!WARNING]
> **代码对齐须知**：本方案中涉及的数据结构、事件定义和接口必须与以下源文件保持一致：
>
> - 食材数据模型：`inventory.hpp` (采用 `IngredientType` 和 `IngredientBatch` 分离的批次管理模型)
> - 系统事件：`system_events.h` (`sys_event_type_t` 枚举)
> - 食材分类：`category_table.c` (11 个分类：水果/蔬菜/肉禽/海鲜/蛋奶/豆制品/主食/调味品/饮品/零食/冻品/其他)
> - GUI 桥接接口：`gui_bridge.h` (gui\_bridge\_xxx 系列函数)

***

### 1.2 全局组件（所有页面共享）

#### 1.2.1 状态栏 (Status Bar)

位于屏幕顶部，高度 50px，全局常驻（全屏场景可隐藏）。

| 位置 | 内容            | 说明                                               |
| -- | ------------- | ------------------------------------------------ |
| 左侧 | 设备名称          | `"小鲜智能冰箱"`，静态文本                                  |
| 左侧 | 语音助手状态图标      | 唤醒词检测中 / 聆听中 / 播报中，动态切换                          |
| 右侧 | Wi-Fi 图标 + 状态 | `LV_SYMBOL_WIFI`，彩色 = 已连接，灰色 = 离线；点击可进入 Wi-Fi 设置 |
| 右侧 | 时间 HH:MM      | RTC 同步后显示当前时间，否则显示 `"--:--"`                     |
| 右侧 | 日期 M月D日       | RTC 同步后显示（如 `6月16日`），否则隐藏                        |

状态栏背景色：白色 `#FFFFFF`，底部带 1px 浅灰分割线 `#E0E0E0`。

#### 1.2.2 全局通知弹窗 (Notification Overlay)

一个模态对话框组件，悬浮在所有页面之上：

- **触发来源**：系统事件（如临期预警 `gui_bridge_show_notification`）、操作反馈（存入/取出成功）
- **类型**：
  - `info` — 普通信息，蓝色图标，自动 3 秒消失
  - `warning` — 警告，黄色图标，需用户点击"知道了"关闭
  - `error` — 错误，红色图标，需用户确认
- **布局**：居中卡片，顶部图标 + 标题 + 正文 + 底部确认按钮

#### 1.2.3 语音交互指示器 (Voice Indicator)

底部悬浮条，宽 600px、高 80px，圆角 16px，深灰色带 80% 透明度背景（`LV_OPA_80`），配合底层阴影实现现代悬浮感（避免使用高斯模糊以节省性能）：

| 状态        | 显示内容                 | 颜色 |
| --------- | -------------------- | -- |
| 空闲 (Idle) | 隐藏                   | —  |
| 监听中       | `聆听中...`          | 蓝色 |
| 识别中 (处理中) | `正在理解...`          | 黄色 |
| 播报中 (TTS) | 显示当前播报文本（滚动字幕）       | 绿色 |
| 执行结果      | `已添加：苹果 x2`（2秒后消失） | 绿色 |
| 错误        | `错误`           | 红色 |

#### 2.2.1 左侧核心信息区（约 320px 宽）

- **天气组件**（上半部分，约 120px）：
  - 天气图标（50px）+ 温度数字（font\_cn\_36）+ 城市名（font\_cn\_18）
  - 温度：粗体 `#333333`，如 `28°C`
  - 城市/天气描述：`#888888`，如 `北京 · 晴`
  - 天气数据来源：云端 API（Wi-Fi 连接时），离线则显示 `"-- · --"`
- **留言板组件**（下半部分，约 200px）：
  - 白色卡片，圆角 16px，阴影
  - 标题栏：`"📋 留言板"`，font\_cn\_18
  - 内容区：显示最新留言文本（如家庭留言），font\_cn\_18，`#888888`
  - 无留言时显示 `"暂无留言"`，浅灰
  - 留言来源：语音录入（"小鲜，留言 XXX"）或手机 Web 面板推送
  - 最多显示 3 条，超出可滚动

> \[!NOTE]
> **待实现后端**：天气组件需要新增云端天气 API 调用模块；留言板需要新增本地存储和 Web 面板推送接口。当前代码中均无对应后端实现。

#### 2.2.2 右侧应用图标网格（App Grid，约 680px 宽）

- **布局**：4 列 × 2 行，支持横向翻页
- **当前实现（5 个应用）**：

| 图标 | 名称   | 点击行为                | 实现状态   |
| -- | ---- | ------------------- | ------ |
| ≡ (List) | 库存管理 | 进入库存应用（查看/管理食材）     | 基础版已实现 |

| ⚙️ (Set) | 系统设置 | 进入设置页面              | 待实现 |
| 🎙️ (Mic) | 语音助手 | 激活语音对话模式，显示全屏语音交互界面 | 新增     |
| 🛒 (Cart)| 购物清单 | 进入购物清单（根据缺货自动生成）    | 未来扩展   |

> **注**：为保证渲染稳定性，界面图标已统一从彩色 Emoji 迁移为 LVGL 内置的 FontAwesome 矢量图标 (`LV_SYMBOL_*`)。

- **单个图标的卡片规格**：
  - 尺寸：150 × 130px
  - 背景：白色 `#FFFFFF`，圆角 `radius: 16px`，阴影 `0px 4px 12px rgba(0,0,0,0.08)`
  - 图标：占卡片 60% 高度，使用 LVGL 内置符号或自定义字体图标
  - 名称：居中，font\_cn\_18，`#333333`
  - 按压态：缩放至 95%，背景变为 `#F0F4FF`

#### 2.2.3 拍照存取快捷入口（底部通栏）

- 位于桌面底部，横跨全宽，是一个**视觉突出的横幅按钮**
- 高度：80px
- 背景：主色调渐变（`#4A90E2` → `#357ABD`），圆角 12px，左右 margin 30px
- 图标：大号相机图标 `📷` + 文字 `"拍照存取"`，白色，font\_cn\_24，粗体
- 副文字：`"对准食材拍照，自动识别并存入"`，font\_cn\_16，白色半透明
- 点击行为：直接进入拍照录入页面（见 2.6.1）
- 区别于普通应用图标：更宽、更鲜艳、固定在底部，作为最高频操作的快捷入口

#### 2.2.4 翻页机制

- 若应用数量超过 8 个（4 列 × 2 行），自动分页
- 页面底部显示圆点指示器（当前页高亮）
- 支持左右滑动手势翻页

#### 2.3.1 侧边栏 (Left Sidebar, \~200px 宽)

| 导航项    | 图标               | 点击行为           | 红点/角标         |
| ------ | ---------------- | -------------- | ------------- |
| ← 返回桌面 | `LV_SYMBOL_HOME` | 返回 Launcher 桌面 | —             |
| 全部食材   | `LV_SYMBOL_LIST` | 显示所有库存食材（默认视图） | —             |
| 临期预警   | `LV_SYMBOL_BELL` | 筛选 3 天内过期食材    | **红色圆点 + 数量** |
| 水果      | -                | 筛选 category == "水果" | —             |
| 蔬菜      | -                | 筛选 category == "蔬菜" | —             |

- 当前选中项：左侧蓝色竖线高亮 (4px) + 文字变蓝

#### 2.3.2 食材卡片 (Ingredient Card)

每个食材以卡片形式展示在主内容区（Flex 流式布局，自动换行）。

- **卡片尺寸**：240 × 170px（横屏更宽，卡片可稍大）
- **卡片内容**：

| 区域 | 内容                           | 样式                                                             |
| -- | ---------------------------- | -------------------------------------------------------------- |
| 左上 | **食材名称**（如"苹果"），font\_cn\_24 | `#333333`，粗体                                                   |
| 右上 | **分类标签**（如"水果"），font\_cn\_18 | `#4A90E2`，蓝色胶囊背景                                               |
| 中间 | **总数量** `x 3`，font\_cn\_18   | `#888888`                                                      |
| 左下 | **食用建议**，font\_cn\_18        | 正常：🟢 `"建议 7 天内食用"`；临期 ≤ 3 天：🟡 `"建议 2 天内食用"`；已过期：🔴 `"尽快食用！"` |
| 右下 | **操作按钮** `+` / `-`           | 圆形按钮，40px，点击增减数量                                               |

- **过期文案规则**：
  - 已过期 (remaining < 0)：`"尽快食用！"`，红色
  - 临期 (remaining ≤ 3)：`"建议 X 天内食用"`，黄色
  - 正常 (remaining > 3)：`"建议 X 天内食用"`，绿色
- **点击卡片**：进入食材详情页（含批次明细，见 2.3.3）
- **空状态**：冰箱为空时，中央显示 `"冰箱空空如也，去买点东西吧！"`（font\_cn\_18，`#888888`）+ 一个大插图

> \[!IMPORTANT]
> **关于批次管理**：数据模型已重构，采用 `IngredientType`（食材种类）+ `IngredientBatch`（单次存入批次）的架构，支持 FIFO 批次存取。详情页将展示按批次倒序排列的明细，自动先消耗最早存入的批次。

<br />

> - **消耗速率**：通过 `compute_consumption_rate(name, 7)` 获取近 7 天的日均消耗量
> - **"取出"按钮**：调用 `remove_ingredient(name, 1)`
> - **"存入"按钮**：进入拍照/语音录入流程

### 2.4 智能食谱应用 (Recipes App)

**整体布局**（与库存应用相同的侧边栏模式，横屏）：

```

## 3. 视觉与中文字体方案 (Chinese Typography)

必须完美支持中文显示是产品级 GUI 的基础。

### 3.1 字体生成策略

- 彻底弃用 LVGL 默认的简陋 `simsun_16_cjk` 字库（字数太少容易出现方块字）。
- 选择一款现代化开源中文字体（如 **阿里巴巴普惠体** 或 **思源黑体 Noto Sans SC**）。
- 使用 LVGL 的 FreeType 矢量字体引擎 (`lv_freetype`)，直接在 SD 卡加载精简的 `.ttf` 字体文件，动态渲染所需字号，避免生成巨大的 `.c` 静态字库占用 Flash。。
- 为了建立清晰的排版层级，生成以下字号：

| 字体别名           | 字号   | 用途           |
| -------------- | ---- | ------------ |
| `font_cn_16`   | 16px | 辅助文字、标签、脚注   |
| `font_cn_18`   | 18px | 正文、卡片内容、按钮文字 |
| `font_cn_24`   | 24px | 页面标题、卡片标题    |
| `font_cn_36`   | 36px | 大时钟、关键数值     |
| `font_icon_24` | 24px | LVGL 内置符号图标  |

│   ├── gui_app.cpp            // 初始化 & 屏幕切换管理器
│   ├── gui_styles.cpp         // 颜色 & 通用样式定义
│   ├── gui_launcher.cpp       // 桌面网格 & 应用图标渲染
│   ├── gui_standby.cpp        // 待机/锁屏界面
│   └── apps/

        └── NotoSansSC-Regular.ttf    // SD 卡中的 TTF 字体文件，由 lv_freetype 运行时加载
        (注：不使用 .c 静态位图字库，采用 FreeType 引擎动态渲染)
```

### 4.1 屏幕切换管理器 (`gui_app.cpp`)

采用 **LVGL Screen 栈** 管理模式：

```cpp
// 核心 API
void gui_app_init(void);              // 初始化，显示 Splash → Launcher
void gui_app_navigate_to(app_id);     // 导航到指定应用
void gui_app_go_back(void);           // 返回上一级（出栈）
void gui_app_show_standby(void);      // 进入待机
void gui_app_wake_from_standby(void); // 从待机唤醒
```

- **Launcher 常驻内存**：作为高频入口，桌面加载后不销毁。
- **按需加载与销毁**：其他应用页面按需创建 `lv_obj_t *screen`，切换回桌面时调用 `lv_obj_delete` 释放内存。
- 维护导航栈用于 `← 返回桌面` 逻辑。

### 4.2 与 `gui_bridge` 的关系

```
system_events → gui_bridge (抽象层) → gui_app (LVGL 实现)
```

- `gui_bridge` 负责业务抽象，并使用 `lv_async_call()` 或 LVGL 消息队列将后台事件安全地推送到 LVGL 主渲染线程执行，彻底避免死锁。
- `gui_app` 负责具体的 LVGL 控件渲染
- 所有跨 Task 的 GUI 操作必须通过 `gui_bridge` → `gui_app` 路径

## 5. 交互流程图

### 5.1 核心用户旅程：食材存入

```
用户拿食材靠近冰箱
    │
    ▼
PIR 检测 → 屏幕唤醒（若在待机）
    │
    ▼
┌─────────────────────────────────────────────────────┐
│ 方式1: 桌面底部 → 点击"拍照存取" → 拍照 → AI识别 → 确认存入  │
│ 方式2: 库存应用 → 底部 📷 → 拍照 → AI识别 → 确认为新批次     │
│ 方式3: 唤醒词 → "小鲜，存入苹果三个" → 语音识别 → 确认存入     │
│ 方式4: 桌面 → 语音助手 → 全屏对话 → 确认存入                │
└─────────────────────────────────────────────────────┘
    │
    ▼
存入成功通知 → 作为新批次加入库存 → 自动刷新库存列表
```

### 5.2 核心用户旅程：食材取出

```
用户打开冰箱取食材
    │
    ▼
库存应用 → 找到食材卡片 → 点击 [-] 按钮
    │
    ▼
FIFO：自动从最早批次扣减 1 个单位
    │
    ├── 该批次仍有剩余 → 更新批次数量，总数量 -1
    └── 该批次归零 → 移除该批次，自动切换到下一个最早批次
    │
    ▼
取出成功通知 → 自动刷新库存列表
```

### 5.3 核心用户旅程：临期提醒

```
后台定时任务 (每小时)
    │
    ▼
check_expiring_ingredients(3)
    │
    ├── 无临期 → 无操作
    │
    └── 有临期 →
            ├── gui_bridge_show_notification("临期提醒", "有3种食材需要尽快处理！")
            ├── WebSocket 推送至手机 Web 面板
            └── 待机界面底部显示临期条
    │
    ▼
用户点击通知 → 跳转库存应用 → 临期预警视图
```

### 5.4 核心用户旅程：食谱推荐

```
Launcher 桌面 → 点击"智能食谱"
    │
    ▼
食谱应用 (推荐食谱视图)
    │
    ├── 自动显示按库存覆盖率排序的食谱
    │
    ├── 点击"今天吃什么" 🎲 → 随机推荐 + TTS 播报
    │
    ├── 点击"联网推荐" → 云端 LLM → 新食谱加入本地库
    │
    └── 点击某个食谱卡片 → 详情页
            ├── 查看所需食材 (绿色=齐全, 红色=缺少)
            ├── "联网获取详细做法" → 云端返回详细步骤 + TTS 播报
            └── "加入购物清单" → 缺失食材自动添加
```
