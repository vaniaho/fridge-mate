## 1.1 核心功能：食材管理

### 1.1.1 食材存入和取出

- (1) **图像识别**（**本地为主，端云协同**）
  - \==本地部署图像分类模型==，实现手持食材的本地识别
  - 端云协同策略（可以参考）
    - 本地置信度 ≥ 60%：显示 Top-3 候选，用户触控选择名称和数量或启用云端模型
    - 本地置信度 < 60%：自动上传图片至云端模型识别并返回结果
    - 未联网：始终本地决策
- (2) **语音识别**（**本地唤醒，云端识别**）
  - 用户口述录入/取出的食材类别和数量，将语音上传至云端模型，云端完成识别并返回特定格式的操作信息，本地食材库进行修改
- (3) 手动操作：在本地食材页面，手动点击“+”“-”按键实现存入和取出
- *(4) 扫码识别：对于带包装的食物，通过扫描条形码实现录入*

### 1.1.2 基于本地食材库的智能助手

- 本地功能
  - **数据存储与查询**
    - 当前仓库中的食材名称、类别、数量、存入时间等信息
    - 一定时间内食材存入、取出的历史记录
    - 每种食材的消耗速率
  - **食材过期提醒**
    - 根据每种食材的保质期和已存放时间，提醒用户尽快处理即将过期的食物
  - **智能补货建议**
    - 根据食材当前库存和过往消耗速率，在即将耗尽或已耗尽时提醒用户补货
  - **食谱匹配与推荐**
    - 根据**本地食谱库**与食材库，检测当前完全可做（食材齐全）和部分可做（缺 1-2 种食材，标红缺失食材并提示）家常菜
- 云端功能
  - **联网食谱推荐**
    - 上传本地食材库至云端模型，返回特定格式的推荐食谱，并能将食谱保存至本地食谱库中
  - 其他依赖云端模型的特定功能（待补充）
  - 与云端大模型直接语音交互

## 1.2 语音交互

### 1.2.1 本地语音唤醒

- 如 “小鲜小鲜”，激活语音助手

> 当前状态：`audio_hal` 已接入 `wake_word_task` 与 `wake_word_engine` 适配层，优先加载 ESP-SR WakeNet 模型。空闲时采集 16 kHz 麦克风 PCM，ASR/实时语音期间暂停；TTS 播放期间可继续监听，命中后先停止播报、等待播放状态收敛，再投递 `EVT_WAKE_WORD_DETECTED`。设置页和 Web API 已支持唤醒开关、灵敏度、TTS 中打断开关和连续对话窗口。正式“小鲜小鲜”唤醒仍需刷入对应 WakeNet 模型。

### 1.2.2 本地识别简单命令词

| 命令词（参考） | 动作               |
| ------- | ---------------- |
| 拍照存取    | 触发食材存入/取出的图像识别方式 |
| 语音存取    | 触发食材存入/取出的语音识别方式 |
| 撤销      | 撤销上次存入、取出操作      |
| 库存      | 打开库存界面，播报库存摘要    |
| 今天吃什么   | 触发食谱推荐           |
| 当前温度    | 播报当前温度           |
| 设置      | 进入系统设置           |

### 1.2.3 云端大模型识别复杂命令

- 本地命令词以外的指令，上传到云端大模型
  - 若识别为功能操作，让云端模型按提前设定好的格式将指令返回本地并执行
  - 若不是功能操作，则返回正常的 AI 回复文本和语音，在本地播报

> 当前状态：已实现异步 LLM worker、OpenAI 兼容 SSE 流式输出、库存上下文注入和库存写操作二次确认。简单“放入/取出”语音文本会优先在本地解析，不必每次走云端 LLM。

> 连续对话：板载语音在 TTS 播完后会按设置进入免唤醒追问窗口，默认 10 秒；确认/取消类流程即使关闭普通连续对话，也会保留最短确认窗口，避免写操作确认被提前终止。

### 1.2.4 TTS （文本转语音）

参考方案：

| 层级          | 覆盖内容             | 实现方式            | 存储位置            | 触发条件     |
| ----------- | ---------------- | --------------- | --------------- | -------- |
| **Layer 1** | 高频固定短语           | 预录音 PCM 片段      | SD 卡 / Flash    | 始终本地     |
| **Layer 2** | 食材名称、数量词         | 拼音拼接合成（Ekho/余音） | SD 卡（\~8MB 音素库） | 始终本地     |
| **Layer 3** | 云端 LLM 生成的菜谱、建议等 | 云端 TTS API 流式返回 | —               | WiFi 可用时 |

> 当前状态：代码已优先落地 Layer 3 云端 TTS（火山双向 WebSocket，16 kHz PCM 播放）。Layer 1/Layer 2 本地预录音或拼音音素库尚未实现。

## 1.5 数据持久化与时钟

### 1.5.1 RTC 电池座

板上提供 RTC 电池座（支持 3V\~3.3V 可充电电池，尺寸 1220），本项目使用该电池保持系统时钟：

- 断电后保质期倒计时不重置
- NTP 对时（联网时）+ RTC 保持（离线时）双重保障
- 食材出库时记录精确时间戳，用于消耗速率分析

### 1.5.2 Micro SD 卡槽

用途：

| 数据                  | 存储内容                    |
| ------------------- | ----------------------- |
| 预录音 TTS 片段（Layer 1） | \~2MB PCM 文件            |
| 拼音音素库（Layer 2）      | \~8MB（或更换为 30 个食材名称预录音） |
| 食材库存数据库             | SQLite / JSON，< 1MB     |
| 传感器历史日志             | CSV，每天追加，定期轮转           |
| 系统日志                | 调试用，可配置关闭               |
