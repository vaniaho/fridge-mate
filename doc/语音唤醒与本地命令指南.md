# 语音唤醒与本地命令指南

本文记录当前本地唤醒、播音打断、连续对话和本地命令词的实现边界，便于后续把工程从“WakeNet 框架可用”推进到“真实产品体验可用”。

## 当前实现

- `audio_hal` 常驻 `wake_word_task`，空闲时读取 ES7210 麦克风，优先使用 ESP-SR WakeNet。
- ASR、端到端实时语音期间，唤醒任务会暂停，避免多个任务同时占用麦克风。
- 普通 TTS 播放期间可继续监听。命中唤醒词后会停止 TTS，等待播放状态收敛，再投递 `EVT_WAKE_WORD_DETECTED`。
- 运行时语音设置保存在 NVS：
  - 本地唤醒开关
  - 唤醒灵敏度
  - TTS 播放中是否允许唤醒打断
  - TTS 后连续对话窗口，默认 10 秒
- Web API：`GET/POST /api/voice/settings`。
- 屏幕设置页：语音模型页顶部提供同样的运行时设置。

## 训练或获取“小鲜小鲜” WakeNet 模型

ESP-SR 的 WakeNet 推理接口已经接入，但项目内没有自定义 WakeNet 训练流水线。真实“小鲜小鲜”模型通常有三种路径：

1. 使用 ESP-SR 已发布的相近唤醒词模型。
   当前默认是 `wn9_nihaoxiaozhi_tts`，可用于验证唤醒链路，但不是“小鲜小鲜”。

2. 通过 Espressif 的 ESP-SR 模型定制渠道生成自定义 WakeNet。
   建议准备覆盖男女老少、远近距离、厨房/冰箱环境噪声、TTS 播放回声场景的数据。关键词建议固定为“小鲜小鲜”，负样本要包含“小新”“小仙”“新鲜”等近音词。

3. 临时方案：使用云端/离线 KWS 或自定义 TFLite Micro 关键词模型。
   这种方式需要新增后端适配，不建议和当前 WakeNet 后端混在同一个模型分区里。

数据建议：

- 正样本：至少数百到数千条“小鲜小鲜”，覆盖不同说话人、语速、距离和设备摆放。
- 负样本：日常厨房对话、TTS 播报音频、电视/手机人声、近似音词。
- 采样格式：16 kHz、16-bit、mono PCM/WAV，尽量贴近设备麦克风链路。
- 评估指标：安静环境召回、噪声环境召回、TTS 播放中误触发、连续 24 小时误唤醒次数。

## 刷入 WakeNet 模型

当前分区表包含：

```text
model, data, spiffs, , 3M
```

推荐流程：

1. 将 ESP-SR 生成的 WakeNet 模型放入 `managed_components/espressif__esp-sr` 可识别的模型目录，或按 ESP-SR 文档配置模型路径。
2. 修改配置：
   - `CONFIG_MODEL_IN_FLASH=y`
   - 选择对应 `CONFIG_SR_WN_*` 模型项
   - `CONFIG_SMART_FRIDGE_WAKE_WORD_MODEL_NAME="实际模型名"`
3. 重新生成 `build/srmodels/srmodels.bin`。
4. 单独刷入模型分区，或随整机固件一起烧录。

示例命令形态：

```powershell
idf.py partition-table
idf.py flash
```

如果只更新模型分区，可根据构建输出中的 `model` 分区地址刷入 `build/srmodels/srmodels.bin`。

注意事项：

- `CONFIG_SMART_FRIDGE_WAKE_WORD_MODEL_NAME` 必须和模型分区内的 WakeNet 名称匹配。
- 模型分区大小目前是 3 MB，多个 WakeNet 或未来本地命令模型可能需要继续扩大。
- Windows 下 ESP-SR `movemodel.py` 读取 `sdkconfig` 的编码问题已通过 CMake patch 自动修复。

## 扩展本地命令词

当前“本地命令”不是语音声学模型，而是 ASR 文本后的轻量解析，主要用于绕过云端 LLM 的简单库存操作。

主要入口：

- `components/audio_hal/audio_hal.cpp`：`is_local_business_command()` 决定端到端实时语音中哪些文本交给本地业务处理。
- `components/ai_agent/response_parser.cpp`：解析“放入/取出/确认/取消”等库存指令。
- `components/ai_agent/response_executor.cpp`：执行或等待确认。

扩展建议：

1. 先扩展 ASR 文本关键词，而不是直接训练声学命令词。
2. 每个新意图都要定义：
   - 触发表达，例如“加入购物清单”“提醒我买牛奶”
   - 槽位，例如食材名、数量、单位、时间
   - 是否需要确认
   - 失败时的 TTS 提示
3. 对高风险写操作保持二次确认。
4. 对“确认/取消”保持上下文相关，只作用于最近一次待确认操作。

建议下一批本地命令：

- “加入购物清单 牛奶两盒”
- “今天有什么快过期”
- “冰箱里还有什么水果”
- “取消刚才的操作”
- “把音量调低一点”

## 实机调参建议

- 安静环境误触发高：降低灵敏度，或把 TTS 中打断关闭。
- 播音中误触发高：保留 TTS 打断开关，但提高 TTS 命中次数或 SNR 阈值。
- 唤醒不灵：提高灵敏度，确认模型名和 `model` 分区是否匹配。
- 连续对话容易误收尾：把连续窗口调到 10 到 15 秒。
- 连续对话太打扰：调到 0 秒，只保留明确需要确认的流程。
